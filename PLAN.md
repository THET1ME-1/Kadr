# 🎬 Kadr — план разработки

**Kadr (Кадр)** — трекер просмотренных фильмов и сериалов. Material 3 Expressive,
крупный необычный дизайн, статистика и кастомизация на максимуме.
ДНК инфраструктуры — из **ScoreMaster** (тема, i18n, настройки, синк).

> Легенда: `[x]` сделано · `[~]` частично · `[ ]` в планах

---

## 🎨 Дизайн-направление (утверждено)

- **Material 3 Expressive** — крупные формы, выразительная типографика, пружинные
  анимации, «shared-axis» переходы между экранами (уже в теме).
- **Шрифты**: Unbounded (заголовки/крупные цифры) + Onest (текст), кириллица + латиница.
- **Кнопки** — закрашенные (`FilledButton`), широкие, со скруглением «таблетка»
  (`StadiumBorder`, высота 56). Уже зашито в теме.
- **Все попапы — снизу** (`showModalBottomSheet`, скругление сверху 28, ручка).
  Никаких центральных диалогов.
- **Иконки** — бесплатные M3 (`Icons.*_rounded/_outlined`), без платных наборов.
- **Оценки — настроения** (emoji-mood, как в референсе: Ужасно → Восхитительно),
  а не только сухие звёзды. Цветовая шкала красный→зелёный.
- **Кастомизация**: тёмная/светлая/системная/авто-по-времени, Material You (цвет
  из обоев), AMOLED, живой колор-пикер (HSV-колесо + HEX), 8 палитр.
- **Крупные постеры**, сетка/список переключаемо (кнопка «Стиль» в AppBar).
- **Группировка просмотров по месяцам** («Просмотры октябрь 2023 г.»), как в референсе.

### Экраны (по референсу + улучшения)
1. **Оболочка** — 4 вкладки снизу: Буду смотреть / Просмотрено / Обзор / В кино
   + выезжающее меню (Главная, Поиск, Статистика, Списки, Настройки, О приложении).
2. **Карточка фильма** — герой-бэкдроп, постер+название+год+хронометраж+слоган,
   рейтинг + твоя оценка-настроение, жанры; ряд действий (Просмотрено / Списки /
   Буду смотреть / Другое / Избранное); описание, режиссёр, бюджет/сборы, тест
   Бекдел, внешние ссылки (Кинопоиск/IMDb/TMDb/Rotten Tomatoes/Trakt/Wikipedia).
   Вкладки ИНФО / ОТЗЫВЫ (гистограмма рейтинга 1–10).
3. **«Когда посмотрели?»** — нижняя панель (Неизвестная дата / Только что завершил /
   Сегодня в… / Вчера в… / Выберите дату). Несколько просмотров на фильм.
4. **Поиск** — по TMDB, живой, с постерами.
5. **Статистика** — крупная, богатая (см. раздел ниже).
6. **Списки** — свои списки с иконкой/цветом/счётчиком (Избранное, Мультфильмы, 18+, …).
7. **Настройки** — внешний вид, язык (7), данные/синк, о приложении.
8. **Фильтры/Сортировка/Стиль** — по году, жанру, актёрам, просмотрено-или-нет, +свои.

## 🎞 База фильмов — решение: **kinopoisk.dev (ПоискКино API)**

TMDB стал платным → перешли на **kinopoisk.dev** (домен API: `api.poiskkino.dev`).
Для РФ-аудитории даже лучше:
- ✅ **Русские названия и постеры** из коробки (то, что нужно).
- ✅ Рейтинги Кинопоиск + IMDb, хронометраж, жанры, актёры, бюджет/сборы, сериалы.
- ✅ Поиск по названию (в т.ч. по оригинальному англ.) + фильтры (жанр/год/рейтинг).
- ⚠️ **Демо-тариф: 200 запросов/сутки** → агрессивное кэширование, ленивая
  фоновая дозагрузка (сначала то, что видно), результаты хранятся в библиотеке.
- Ключ: `lib/config/api_config.dart` (+ override `--dart-define=KINOPOISK_KEY=…`).
  Матчинг: по совпадению оригинального названия + года (высокая точность).

**Дополнительно (опц.):** OMDb (Rotten Tomatoes/Metacritic), Trakt (гистограмма оценок).

## 💾 Данные, синхронизация и бэкап (как в ScoreMaster)

- **Локально-первичное** хранилище: слой `MovieRepository` (`ChangeNotifier`) поверх
  `Store` (SharedPreferences сейчас; при росте объёма — миграция на Hive/Isar).
  Кэш метаданных фильмов из TMDB + пользовательские просмотры/оценки/списки/заметки.
- **Бэкап JSON**: экспорт/импорт всех данных (`file_picker` + `share_plus`) —
  ровно как в ScoreMaster (не OAuth, чтобы не хранить секреты).
- **Синхронизация между устройствами**: переносится готовый слой ScoreMaster —
  `services/sync/` (WebDAV, P2P, папка-бэкап, `sync_merge`). Подключим после MVP.
- В бэкап входят: списки, просмотры (даты+оценки), заметки, оценки-настроения,
  избранное, И настройки (тема/цвет/язык/стиль).
